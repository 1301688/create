# OS とは

<br />{" "}

- 基本ソフトウェア
- オペレーティングシステム
- コンピュータのオペレーション（操作・運用・運転）を司るシステムソフトウェア　コンピュータのスーパーバイザー
- OS はアプリケーションソフトウェアを動作させるのが第一の目的
- アプリケーションプログラムとハードウェアの中間に位置し、
- アプリケーションプログラムとの I/O インターフェースを提供すると同時に
- ハードウェアなどの各リソースに対して効率的な管理を行う

<br />{" "}

# OS の役割

<br />{" "}

- ハードウェアの抽象化
- 幾つかあるハードウェアに対して、抽象化された利用方法を提供することでソフトウェアの開発を容易にしている

<br />{" "}

- リソースの管理
- 複数のアプリケーションソフトウェアを同時に利用する際に、互いに独立して動作できるようにリソースを管理している

<br />{" "}

- コンピュータの利用効率の向上
- 複数のタスクを同時に実行する際に、リソースの割り当ての順番や処理の割り当て時間を工夫することで、
- 全体のスループット（処理能力、効率）を向上させる

<br />{" "}

# OS の構成

<br />{" "}

| 名称             | 主な機能                                                          | 説明                                                                                                             |
| ---------------- | ----------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------- |
| ブートローダー   | コンピュータ起動時、OS を起動させるプログラム                     | 電源投入 →BIOS 起動 → ブートローダー起動 →OS 起動                                                                |
| カーネル         | OS の中核となるプログラム　アプリケーションとハードウェアの架け橋 | 一番ハードウェアよりで頑張っているソフトウェア                                                                   |
| デーモン         | 常駐プログラム　バックグラウンドプロセス　常に動作するプロセス    | UNIX 系、Mac=デーモン　 Windows=サービス                                                                         |
| シェル           | 言語をコンピュータに伝えるプログラム                              | UNIX 系、Mac= ターミナル、Windows=コマンドプロンプト                                                             |
| デスクトップ環境 | GUI であるデスクトップ環境を提供するプログラム                    | Windows、Mac は OS とセット　 Linux は OS と別でインストール                                                     |
| アプリケーション | ファイルシステム、ブラウザ、設定機能、メールなどの標準 App        | OS（カーネル）だけでは基本的な機能のみのため、一般のユーザが使いやすいよう様々なソフトウェアがセットになっている |

<br />{" "}

# OS の機能

<br />{" "}

| API、ABI | アプリケーションソフトウェアを動作させるインターフェース |
| プログラム実行 | プロセス（実行中のプログラム）の生成から終了 |
| 割り込み | コンピュータのハードウェアまたは実行中のプログラムが発生させる急事タスクへのメモリ割り当て |
| モード | OS のカーネルが使用するスーパーバイザーモード（ハードウェアに無制限にアクセス）と OS のカーネル以外がほぼ使用するユーザーモード（アプリケーションにアクセス） |
| メモリ管理 | 物理メモリと仮想メモリ　スワッピング　 RAM とストレージ |
| タスク管理 | タスクへのメモリ割り当て（実行）とメモリ開放（片付け）|
| ファイル管理 | コンピュータはファイルという形でディスクにデータを格納する　この格納するプログラムをファイルシステムといいファイル管理する |
| 周辺機器（デバイス）管理 | デバイスドライバというハードウェアとのやり取りをするソフトウェアで周辺機器を管理する |
| ネットワーク | TCP/IP プロトコルをサポートしている |  
| セキュリティ | 処理を許可すべき要求と処理すべきでない要求を識別する「認証」とさらに「監視」をしている|

<br />{" "}

# まとめ

<br />{" "}

- コンピュータ上で起こるプログラムを管理する基幹ソフトウェア
- コンピュータの人格にあたるもの
- サーバーのもつ能力を管理するソフトウェア

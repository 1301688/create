# OS の種類と特徴の理解その 1

<br />{" "}

- OS にはサーバー OS とクライアント OS がある
- 以下、マトリックスまとめ

| ----                   | クライアント OS                                     | サーバー OS                                                      |
| ---------------------- | --------------------------------------------------- | ---------------------------------------------------------------- |
| 概要                   | ローカルの単一サーバー専用として開発された OS       | リモートの大きいサーバー 専用として 開発された OS                |
| 目的                   | 主にユーザーが使用することが目的                    | クライアント OS の使用を補助・効率化するために使用することが目的 |
| 構成                   | 単一だけで使用する                                  | 複数で共有して使用する                                           |
| 他のソフト             | Office365, Mail クライアント, ブラウザ              | ミドルウェア(Web サーバー,DB サーバー) , 管理ツール              |
| 代表的なエディション   | windows10                                           | windows server2016                                               |
| 品質, 信頼性           | 低い,単一で使用するため、故障時影響範囲が小さいから | 高い,共有して使用するため、故障時影響範囲が大きいから            |
| クラスタ化             | 通常はなし                                          | 場合によってあり                                                 |
| ディスク構成           | 通常は Raid なし                                    | 通常はミラーリング以上の Raid あり                               |
| CPU                    | 低い                                                | 高い, マルチコア, マルチプロセッサ                               |
| GUI (インターフェース) | 必須                                                | 必須ではない, Unix は OS と GUI はセットではない                 |
| OS サポート            | 通常はなし                                          | 必須, 障害が復旧しないと全サービがストップするから               |
| 形態                   | デスクトップ, ノート, タブレット, モバイル          | ブレードサーバ, ラックマウント, 仮想サーバ                       |

<br />{" "}

# サーバー OS とクライアント OS の分別まとめ

<br />{" "}

- クライアント OS の影響範囲は個人、サーバー OS の影響範囲はグループ
- クライアント OS 上で作ったプログラムを、AWS、GCP、Versel、Netlify などのサービス(サーバー OS を含めた Saas,Iaas)にまかせている
- AWS, GCP, Azzule, Versel, Netlify のサービスはこの辺を Git と連携して解析、プログラムしてくれるイメージ

# OS の種類と特徴の理解その 2

<br />{" "}

- サーバー OS の種類と特徴
- 以下、マトリックスまとめ

| ----             | Windows                                                        | Unix 系                                                                             |
| ---------------- | -------------------------------------------------------------- | ----------------------------------------------------------------------------------- |
| 概念             | 安定性、機能性、企業の基幹システム                             | マルチタスク、マルチユーザー、TCP/IP を開発                                         |
| 操作             | GUI での操作で初心者でも操作しやすい                           | CUI での操作で知識が無いと操作しにくい                                              |
| 使用料金         | 有料、ライセンス料金が必須、サーバーの利用権も必要             | Unix は有償、派生系は一部無償(OSS でライセンス料金が必要ない)                       |
| サポート         | ライセンス料金が必須のため、サポート付き                       | サポートについては有償                                                              |
| よく使われる機能 | 「ActiveDirectory」 でのクライアント管理、社内ファイルサーバー | Apache（Web サーバー用ソフト)、Web アプリケーション開発とオープンソースの組み合わせ |
| メリット         | クライアント OS が windows の場合「AD」で管理がしやすい        | セキュリティが高く、不具合や障害が起こりにくい                                      |
|                  | 入門書、マニュアルが豊富で必要な情報が得やすい                 | 低スペックのサーバーでも動作しやすい                                                |
|                  |                                                                | 処理速度が速い                                                                      |
| デメリット       | 同時接続できるのはユーザー(デバイス)分だけ                     | 操作には専門知識が必要                                                              |
| 形態             | HTTP, FTP, DNS, DHCP, ゲーム, ActiveDrectory                   | HTTP, FTP, DNS, DHCP, ゲーム                                                        |

# サーバー OS まとめ

<br />{" "}

- 専用サーバーや VPS(バーチャルプライベートサーバ、仮想専用サーバ)を利用する際に重要な選択肢となるのがサーバー OS
- 運用に照らし合わせて選択することが必要
- Windows 上でしか動かないアプリやほかの Windows システムとの連携が必要であれば Windows 系 OS
- それ以外なら Unix 系 OS（主に Linux）を選択するのが一般的
- Windows 系 OS と Unix 系 OS はまったく異なる OS のため、同じバイナリ（実行プログラム）を動かすことが基本的にできない
- ユーザー単位ではなく、パソコンを利用する人間全員で共有する
- 多数の PC を同時に接続可能で、大規模オフィスに対応できる
- クライント OS よりも処理が高速で、安定性に優れる
- 業務に支障が出ないよう、障害時のサポートが充実している
- データベースサーバー、WEB サーバーのようなミドルウェアが動作する

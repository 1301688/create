# 今回の目的

<br />{" "}

- これまでなんとなく nodebrew を使って Node.js のバージョンを管理していた
-  少し調べたところ、Node.js のバージョン管理ツールが色々存在しており、何を使用すれば良いか
  わからなかったので、それぞれの比較とできることを理解することが目的
- 比較内容は自分が気になったところだけにする　理由は速くある程度がわかれば良かったから

<br />{" "}

# 使用ケース

<br />{" "}

- OS に Node.js をインストールしたあと、運用を容易にしたいとき

<br />{" "}

# 結論

<br />{" "}

- レポジトリ毎に Node.js が切り替えられるバージョン管理ツールの使用が大半で見られた
- バージョン管理ツールのインストール先がホームディレクトリ・OS 全体（システム）があるが殆どが前者のツールを選択
- 現在は nodebrew を使用していて、困ってはいないが汎用的にしたいのでバージョン管理ツールを変更したいと考えた

<br />{" "}

# 比較基準

<br />{" "}

- Node.js のバージョン管理ツールは「Node.js パッケージ管理ツール npm」「Node.js 実行ツール npx」もセットで管理するので比較基準から除外
- OS によって使いかた、インストール方法が変わる → これは当たり前　 OS によって使える、使えないがある

<br />{" "}

# 基本理解

<br />{" "}

- バージョン管理ツールをインストール後、local コマンドで Node バージョン指定をしたリポジトリには
  プロジェクトフォルダ内に.node-version という設定ファイルができる
  .node-version にはこのプロジェクトで使用する Node.js のバージョンが記述されているため、
  このフォルダに移動した時に、自動的にバージョンが切り替わる仕組みになっている

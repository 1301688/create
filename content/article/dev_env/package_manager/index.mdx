# 今回の目的

<br />{" "}

- パッケージ管理の本質を理解することが目的

<br />{" "}

# パッケージとは

<br />{" "}

- ファイルをひとまとめにした書庫のようなもの　ライブラリ
- ソフトウェアを構成する様々なファイルと、そのソフトウェアが正しく動作するのに必要な命令などが含まれている

<br />{" "}

# パッケージ管理システムとは

<br />{" "}

- オペレーティングシステム (OS) やプログラミング言語というひとつの環境で、各種のソフトウェアの導入と削除、
  そしてソフトウェア同士やライブラリとの依存関係を一元管理するシステム

<br />{" "}

# 依存関係とは

<br />{" "}

- 一つのソフトが複数のソフトを利用するため、「ソフト A をアップデートしたらソフト B、ソフト C が動かなくなった」という状況が発生する
  「ソフト B 5.0 にはソフト A 2.0 以上が必要」といった関係があるため
  これをソフトの依存関係と言う

<br />{" "}

# なぜパッケージ管理するのか

<br />{" "}

- 複数のソフトの依存関係を手動で管理するのは作業コストが高くなり、
  障害リスクも大きいから

<br />{" "}

# パッケージマネージャーを使う

<br />{" "}

- 無数のソフトの依存関係を手動で管理するのは面倒過ぎるので Package Manager（パッケージマネージャー）と呼ばれる
  種類のソフトがよく使われる　 Package Manager は「そのソフトにどのソフトのバージョン何以上が必要か」、
  「今インストールされているソフトは何か」といった情報を持っているため、必要に応じてソフトを勝手にダウンロードしてきて
  インストールして依存関係を管理してくれる

<br />{" "}

- パッケージ管理の種類

<br />{" "}

- OS のパッケージ管理ツール

| OS             | TOOL                          |
| -------------- | ----------------------------- |
| Windows        | Chocolatey , PackageManagemet |
| Mac            | MacPort , Homebrew            |
| Linux          | Linuxbrew                     |
| Linux (RedHot) | rpm , yum                     |
| Linux (Debian) | deb , apt-get                 |

- 言語のパッケージ管理ツール

| langage | TOOL              |
| ------- | ----------------- |
| Ruby    | gem               |
| PHP     | Composer          |
| Python  | pip , Conda       |
| Java    | Maven             |
| Go      | GoMod             |
| Node.js | npm , yarn , pnpm |

<br />{" "}

# まとめ

<br />{" "}

- パッケージ管理システムを使うと、それぞれのソフトウェアの依存関係を管理してくれるので
  手作業でソフトウェアのインストール、設定をせずに開発できる

<br />{" "}
